# Release Evidence

This directory contains canonical templates for M5-T06 through M5-T09. A
template is not release Evidence. The release tooling accepts only reports with
`status: "passed"` and generates a candidate manifest bound to their exact
SHA-256 digests.

The candidate workflow deliberately stops before publication:

1. Run the MVP corpus, offline/security, performance, and package test gates.
2. Materialize passed security and performance reports from the templates.
3. Stage one npm tarball with `prepare-candidate.mjs`.
4. Run the complete release test command through
   `attest-tested-bytes.mjs`; the tarball digest is measured before and after.
5. Validate deterministic pack contents, SBOM, provenance, reports, and tested
   bytes with `check-candidate.mjs`.
6. Optionally copy that exact tarball to a local promotion directory with
   `promote-tested-bytes.mjs`.

Publishing is intentionally outside these tools.

The current canonical reports are generated by:

```sh
node tooling/conformance/run-security-gates.mjs \
  --out docs/compliance/release/SECURITY_REPORT.json
node tooling/conformance/benchmark.mjs \
  --out docs/compliance/release/PERFORMANCE_REPORT.json
```

The performance generator creates and removes its 100,000-file reference
repository in the operating-system temporary directory; generation time is not
included in the warm-discovery measurement.
